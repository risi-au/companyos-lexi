## Access/safety boundaries + gotchas (additions this session)

Prior handoffs' boundaries stand. New learnings:
- Classifier blocked even a **read-only prod docker-logs check** in auto mode — VPS prod
  actions need a plain-chat owner sentence naming the target, or the owner runs it via
  `!` prefix. AskUserQuestion answers still don't count.
- Classifier also blocks `rm -rf` on pre-existing dirs and chained `git stash drop` —
  do destructive cleanup stepwise with the verification output visible before the drop.
- **Merge-race gotcha**: owner merges PRs fast — don't push follow-up commits to a PR
  branch without checking the PR is still open (that's how the #15 sweep got stranded).
- Orca (owner's IDE) PR panel binds to the checked-out branch, not a selected PR.
- Local git identity/gh auth work; owner merges via Orca or github.com.
