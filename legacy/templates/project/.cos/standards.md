# COS Core Standards - Our Way of Working

This file lives in every `.cos/` folder and defines the mandatory patterns.

## Required Files (enforced by onboarding + auditor)
- `README.md` with YAML frontmatter (business, client, project, tags, status, links)
- `decisions.md` (append-only log of key choices)
- `changelog.md` (date-stamped changes)
- `.cos/context.md` (generated by Context Loader - can be .gitignored)
- `.cos/standards.md` (this file or symlink to central version)

## Agent Behavior Rule
**Every agent opening work in this folder must first read:**
1. `.cos/context.md`
2. `README.md` (frontmatter + overview)
3. Relevant subfolder files

Then query gbrain for fresher scoped data if needed.

## Update Protocol
- Significant decision or client change → append to decisions.md + update README last-reviewed + git commit
- Major milestone → trigger Context Loader + gbrain enrichment

## Scoping
All memory operations in gbrain and Hindsight are filtered by the scope defined in frontmatter or folder path. No cross-client bleed allowed.
